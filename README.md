# NornGraph — Система знаний по металлургии никеля ⬡

> Интеллектуальная поисково-аналитическая система на базе **GraphRAG** для работы с корпусом научной литературы по металлургии никеля, меди и платиноидов. Разработано в рамках хакатона **Норникель AI Science Hack** (трек *«Научный клубок»*).

---

## 🏗️ Архитектура системы

```
299 научных документов (PDF, DOCX, статьи)
          ↓ [ETL: populate_graph.py + NotebookLM]
    Neo4j Aura (граф знаний)
    ├── Material, Process, Equipment, Property
    ├── Experiment, Expert, Publication, Condition
    └── Связи: uses, applies, contradicts, describes...
          ↓ [Runtime: server.py + RouterAI]
    FastAPI Backend (порт 8001)
    ├── GET  /api/graph      → граф для vis-network
    ├── POST /api/query      → GraphRAG Q&A (DeepSeek Chat)
    ├── POST /api/upload     → динамическое расширение графа
    └── GET  /api/analytics  → противоречия и пробелы
          ↓
    Web UI (vis-network.js + vanilla JS)
    ├── Интерактивный граф знаний (визуализация сети)
    ├── Чат-интерфейс Q&A (Markdown-ответы + история)
    ├── Аналитика противоречий и пробелов в исследованиях
    └── Загрузка новых документов
```

## 🌟 Ключевые особенности

*   **GraphRAG вместо обычного RAG:** Поиск ведётся по структурированному графу Neo4j, а не просто по векторному индексу. Это даёт точные и проверяемые цепочки рассуждений с защитой от галлюцинаций LLM.
*   **Динамическое расширение графа:** Система умеет принимать новые документы (PDF, TXT, MD) через drag-and-drop прямо в интерфейсе и самостоятельно достраивать новые сущности и связи в Neo4j на лету.
*   **Визуализация противоречий:** Ребра типа `CONTRADICTS` подсвечиваются в графе красным — сразу видно, в каких отчетах или регламентах исследователи расходятся во мнениях.
*   **Гибридный Vision-пайплайн (VLM):** Встроенный загрузчик автоматически извлекает текст через локальный PyMuPDF, но для сложных сканов, таблиц или документов со "сбитыми" шрифтами (например, старые отчеты) доступен режим Vision-распознавания. Он конвертирует страницы в изображения и передает их мультимодальной модели **Qwen2.5-VL-72B**, которая «читает глазами» сложную структуру документа, не теряя ни единого узла.
*   **Компактная и экономичная модель:** Используется `deepseek-chat` (V3) через RouterAI API. Это РФ-совместимо, быстро и в 20–50 раз дешевле проприетарных моделей класса GPT-4 при сохранении высокой точности.

## 🛠️ Стек технологий

| Компонент | Технология | Описание |
| :--- | :--- | :--- |
| **Граф знаний (База данных)** | Neo4j Aura (Cloud) | Облачное хранилище связей и сущностей |
| **ETL извлечение (первичное)** | NotebookLM CLI | Используется для извлечения графа из 1M+ токенов контекста |
| **LLM для GraphRAG** | DeepSeek-Chat (via RouterAI) | Обрабатывает контекст и формирует точные ответы |
| **Backend API** | FastAPI + Uvicorn | Быстрый бэкенд на Python |
| **Frontend UI** | Vanilla JS + HTML5 + CSS3 | SPA-интерфейс, сохранение истории чата в `localStorage` |
| **Визуализация графа** | vis-network.js | Интерактивный рендеринг и поиск по узлам с живым фокусом |
| **Парсинг и OCR (Vision)** | PyMuPDF, Qwen2.5-VL-72B (OpenRouter) | Извлечение текста и интеллектуальный парсинг сканов/изображений |

## 📂 Структура проекта

*   `server.py` — основной FastAPI-сервер (API-эндпоинты GraphRAG, загрузчик документов, аналитика).
*   `populate_graph.py` — ETL-скрипт для наполнения Neo4j из Google NotebookLM через CLI.
*   `bulk_upload.py` — утилита для пакетной или точечной загрузки локальных PDF напрямую на сервер (с опциональным Vision-парсингом).
*   `clear_graph.py` — скрипт для полной очистки базы данных графа.
*   `static/` — файлы веб-интерфейса:
    *   `index.html` — разметка SPA-интерфейса (Markdown рендерится через `marked.js`).
    *   `app.js` — логика работы с API, живая подсветка и фокус на узлах графа при поиске.
    *   `style.css` — стили интерфейса в стиле премиального темного дашборда.
*   `Dockerfile` и `docker-compose.yml` — конфигурация для контейнеризации и удобного деплоя.
*   `.gitignore` — исключение системного мусора, PDF-документов и ключей.
*   `.env.example` — шаблон конфигурационного файла для разработчиков.
*   `requirements.txt` — список Python-зависимостей.

## ⚡ Быстрый старт

### 1. Настройка переменных окружения
Создайте файл `.env` на основе шаблона `.env.example`:
```bash
cp .env.example .env
```
Заполните его вашими ключами:
```bash
ROUTERAI_KEY=sk-... # API-ключ от платформы RouterAI
NEO4J_URI=neo4j+s://xxxxxxxx.databases.neo4j.io # Адрес инстанса Neo4j
NEO4J_USER=your_user
NEO4J_PASSWORD=your_password
NOTEBOOK_ID=your_notebooklm_notebook_id
ROUTERAI_BASE_URL=https://routerai.ru/api/v1
LLM_MODEL=deepseek/deepseek-chat
```

### 2. Запуск через Docker (Рекомендуется для жюри)
Самый простой способ поднять проект — использовать Docker Compose. Все зависимости и окружение подтянутся автоматически:
```bash
docker-compose up -d
```
После успешного запуска приложение будет доступно по адресу: 👉 **[http://localhost:8001](http://localhost:8001)**

### 3. Локальный запуск (Альтернатива)
Если вы хотите запустить проект без Docker, установите зависимости:
```bash
pip install -r requirements.txt
```
Затем запустите FastAPI-сервер:
```bash
python server.py
```

### 4. Наполнение графа (Опционально)
Граф знаний уже хранится в облачной БД Neo4j. Если вам потребуется пересобрать его с нуля с помощью NotebookLM:
```bash
nlm login
python populate_graph.py
```

В репозитории специально оставлены два тестовых файла:
* `data/docs/test_1.txt`
* `data/docs/test_2.txt`

Они содержат искусственно созданные противоречивые научные утверждения о температурных режимах и времени процессов. 
Чтобы проверить, как система автоматически выявляет противоречия и подсвечивает их красными связями (`CONTRADICTS`), просто загрузите оба эти файла по очереди через веб-интерфейс (`drag-and-drop`).
